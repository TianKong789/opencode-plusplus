# OpenCode++ Roadmap

## Phase 0 — Foundation

Goal:
Create the architectural foundation of the platform.

Deliverables:

* Project structure
* Domain models
* Interfaces
* Event definitions
* Configuration framework
* Dependency injection
* Documentation

Success Criteria:

* Project builds successfully
* Core architecture reviewed
* Interfaces finalized

---

## Phase 1 — Execution Runtime

Goal:
Integrate OpenCode through an execution adapter.

Deliverables:

* Execution engine abstraction
* OpenCode adapter
* Workspace management
* Git integration

Success Criteria:

* OpenCode++ can execute a simple task

---

## Phase 2 — Model Intelligence

Goal:
OpenCode++ understands the strengths and weaknesses of a LLM and automatically selects the best model for each task.

Deliverables:

* Model capability profiling
* Task-to-model matching
* Automatic model selection
* Performance tracking per model

Success Criteria:

* System selects optimal model for given task type
* Measurable improvement over naive model selection

---

## Phase 3 — Experience Memory

Goal:
Capture and retrieve engineering experiences.

Deliverables:

* Experience repository
* Reflection storage
* Search capability

Success Criteria:

* Similar experiences can be retrieved

---

## Phase 4 — Skill System

Goal:
Convert successful experiences into reusable skills.

Deliverables:

* Skill extraction
* Skill repository
* Skill versioning

Success Criteria:

* Skills can be reused across tasks

---

## Phase 5 — Multi-Agent Collaboration

Goal:
Introduce specialized agents.

Agents:

* Planner
* Coder
* Reviewer
* Tester
* Researcher
* Reflector

Success Criteria:

* Agents collaborate on task completion

---

## Phase 6 — Evolution Lab

Goal:
Allow prompts, skills, and workflows to evolve.

Success Criteria:

* Experimental assets compete against production assets

---

## Phase 7 — Recursive Self-Improvement

Goal:
Enable benchmark-driven self-improvement.

Success Criteria:

* Measurable improvement across benchmark suites
